Everything in the box

Nine commands. One mental model.

Each subcommand reads from — or writes to — the same local SQLite knowledge store. Combine them in CI, in your editor, or at the prompt. The three highlighted commands are brand-new in v1.1.

📖 Build the index
contextlayer index requires ANTHROPIC_API_KEY

Runs the full Haiku → Sonnet → Opus extraction pipeline against your git log, GitHub PRs, and code. Writes canonical atoms + embeddings + topic clusters. Idempotent — re-runs are near-free.

contextlayer scan requires ANTHROPIC_API_KEY

Code-only ingestion path. For greenfield repos with no PR history — reads manifests, READMEs, and top source files. Day-one conventions on any repo.

🔌 Serve the index
contextlayer mcp keyless

Starts the stdio MCP server. Exposes context_query, context_validate, and context_list_topics to Claude Code, Cursor, Claude Desktop, and any MCP-compatible client.

contextlayer status keyless

Print atom / rule / topic counts and last-indexed time. The fastest "is this thing built yet?" check.

✍️ Capture & export knowledge
contextlayer note keyless

Manually capture a decision atom: contextlayer note "Do not use threading" --scope src/workers/**. No PR? No problem.

contextlayer explain keyless

Renders a Markdown project brief from indexed atoms. Pipe into a fresh LLM session and skip the "explain my project" tax.

contextlayer health

Scores your convention index 0–100 (A–F) across 6 dimensions: atom count, rule count, topic breadth, citation coverage, freshness, and conflict-freedom. Pure read, no API calls.

contextlayer drift

Checks recent commits against indexed rules. Heuristic, deterministic, zero LLM calls — safe to gate CI on. Exits 1 when violations are found. Use in pre-merge hooks. 

$ contextlayer drift --last 10
⚠ 1 potential violation:

  commit def5678 (5d ago)
  ┗━ Violates a_2598: "Do not
     use utils/db_helper"
     Source: PR #8
contextlayer claude-md

Emits a production-quality CLAUDE.md derived from the index — topic-grouped, rules first, citations inlined. --append works for existing files.

Push, not just pull

Catch convention violations before they ship.

context_query is pull: the agent has to ask. context_validate is push: show your proposed change, get a verdict.

The flow
  1. 1.Your LLM proposes a change.
  2. 2.It calls context_validate(proposed_change=...)
  3. 3.A deterministic engine checks scope, anti-patterns, and forbidden phrases — no LLM call needed for the common cases.
  4. 4.Ambiguous cases escalate to a Haiku judge (pennies/call). Premium tier sends every check.
  5. 5.Verdict + cited source_refs come back. The LLM revises.
// Your LLM calls: context_validate(
//   proposed_change="Add endpoint that uses
//                    threading for concurrent DB calls"
// )
{
  "passes": false,
  "violations": [{
    "rule_id": "a_8f1c...",
    "rule_summary": "Do not use threading —
                       caused race condition in #88",
    "why_violated": "Proposal uses threading
                      where async-first is required",
    "severity": "high",
    "source_refs": ["issue:88", "pr:142"],
    "scope": "src/workers/**"
  }]
}
Three modes

Pick how it runs.

A single env var picks the validation mode. All modes share the same local-first query pipeline.

tier · free

Deterministic only

  • ✓ No API key required
  • ✓ Scope, anti-pattern, and forbidden-phrase checks
  • ✓ Sub-15ms warm latency
  • ✓ 100% offline · zero telemetry
tier · hybrid · default

Best of both

  • ✓ Deterministic engine runs first
  • ✓ Escalates to Haiku only on uncertainty
  • ✓ Auto-degrades if no key set
  • ✓ The recommended default for teams
tier · premium

Maximum accuracy

  • ✓ Every check runs through Haiku judge
  • ✓ Highest verdict accuracy
  • ✓ Falls back to deterministic on failure
  • ✓ Best for high-stakes enterprise paths
CONTEXTLAYER_TIER=free|hybrid|premium · query path is keyless on every tier.

Common questions

You're wondering. We'll answer.

Does my code leave my machine?
Never. ContextLayer runs entirely locally. Your code, atoms, and embeddings stay in a SQLite file on your disk. The only network call is to the Anthropic API during the one-time indexing step (using your own key). Queries, validation, and the MCP server are 100% offline.
Do I need to pay?
Querying is free forever — no API key needed. Building an index costs roughly ~$1.50 in Anthropic API credits (your own key). The free validation tier is fully deterministic with zero API calls. Re-indexing is near-free thanks to per-event caching.
Does it work on repos with no PRs?
Yes. Use contextlayer scan to extract conventions from code structure alone — manifests, README, top source files. Or use contextlayer note to capture decisions manually. Both work on any repo, day one.
Which AI tools does it work with?
Any MCP-compatible client: Claude Code, Claude Desktop, Cursor, and others. ContextLayer uses the standard MCP stdio protocol — one .mcp.json block and you're connected.
Does ContextLayer modify my codebase?
No. ContextLayer is read-only. It reads your git history and code files to extract knowledge atoms, but never writes to your repository. The only file it creates is a SQLite database in ~/.contextlayer/.
Can I use ContextLayer in CI?
Yes. The v1.1 commands — health, drift, and claude-md — are keyless, deterministic, and exit-code aware. Wire contextlayer drift into a pre-merge GitHub Action to catch convention violations before they ship, and contextlayer health --json to track index quality over time.
