# demo-data/

This directory holds the data the ContextLayer demo runs against. Two paths:

## Primary: `acme-billing-api/` (synthetic, deterministic)

A 15-PR git repo generated by `build_acme.py`. The PRs deliberately embed:

| PR  | Kind          | Convention / Decision                                |
|-----|---------------|------------------------------------------------------|
| #3  | CONVENTION    | Adopt `Result<T>` for domain errors (not exceptions) |
| #6  | (reinforces)  | Migrate /billing endpoints to `Result<T>`            |
| #8  | DEPRECATION   | Deprecate `utils/db_helper`; use SQLAlchemy session  |
| #11 | DECISION      | Async-first for all I/O routes                       |
| #14 | ANTI-PATTERN  | Never share a SQLAlchemy session across requests     |

The other 10 PRs are ordinary work + 1–3 review comments each — background noise
for the Haiku relevance filter (Stage 1).

This is the **primary demo path** (Appendix A #16 of the design spec): it's
deterministic, bulletproof, and re-indexes in under a minute on any laptop.

**Regenerate at any time:**

```bash
uv run python demo-data/build_acme.py
```

The output (`demo-data/acme-billing-api/`) is gitignored — only the generator
is in version control. Re-running produces the same content; commit SHAs are
pinned via fixed author/date.

## Optional stretch: `fastapi/` (real OSS)

If the Phase 3 stretch goal lands (T+34 → T+36, per `tasks/todo.md`), this
directory will also contain a clone of `tiangolo/fastapi` and its pre-indexed
`.db`. Per Appendix A #16 of the spec, FastAPI is an optional credibility
upgrade — **not** a fallback. The synthetic repo above IS the bulletproof path.

## Pre-indexed SQLite

After the indexer lands at T+12, running `contextlayer index demo-data/acme-billing-api`
writes the index DB to `~/.contextlayer/<repo-hash>/index.db`. For demo
reproducibility we also commit a copy here (`acme-billing-api.db`) at T+24
once the pipeline is locked. `.db` files are otherwise gitignored.
